============================================
C設計レビューAI  CReview v0.4.0
============================================


■ 初回セットアップ
--------------------------------------------

1. config.txt.sample を config.txt にリネーム

2. config.txt を開き API_KEY= の右側に
   ClaudeのAPIキーを貼り付けて保存

   APIキー取得先: https://console.anthropic.com/

3. 完了


■ コードレビュー
--------------------------------------------

方法A: start.bat をダブルクリック
  → 対象ファイルまたはフォルダを入力
  → Enter

方法B: コマンドライン
  creview main.c
  creview src/
  creview main.c util.c parser.c


■ 仕様レビュー
--------------------------------------------

方法A: start_spec.bat をダブルクリック
  → 仕様ファイルを入力
  → Enter

方法B: コマンドライン
  creview --spec 仕様.txt


■ ローカル解析のみ（API不使用）
--------------------------------------------

  creview --local-only main.c

APIキー無しでも使えます。
ローカル静的解析12パターンのみ実行。


■ JSON出力
--------------------------------------------

  creview --format json main.c


■ 出力の見方
--------------------------------------------

【重大】
  クラッシュ可能性あり。最優先確認。

【設計不明】
  仕様曖昧。レビュー推奨。

【保守危険】
  将来バグ化可能。


■ 警告抑制 .creviewignore
--------------------------------------------

プロジェクトルートに .creviewignore を作成

  GLOBAL_OK      グローバル変数警告を抑制
  MACRO_ALLOW    マクロ警告を抑制
  MAGIC_OK       マジックナンバー警告を抑制
  VOLATILE_OK    volatile関連を抑制（将来用）
  PACKED_OK      packed struct関連を抑制（将来用）

  # 以降はコメント


■ 注意事項
--------------------------------------------

- このツールはコード修正しません
- レビュー補助専用です
- 通信先は Claude API のみです
- 外部にコードを保存しません
- 自己更新しません
- API失敗時は「レビュー失敗」とだけ表示します


■ 推奨ワークフロー
--------------------------------------------

1. 実装前    creview --spec spec.txt
2. 実装後    creview src/
3. PR前      creview --local-only src/


■ 終了コード
--------------------------------------------

  0  重大指摘なし
  1  重大指摘あり


■ ファイル構成
--------------------------------------------

  creview.exe          本体
  config.txt.sample    設定テンプレート
  config.txt           設定（自分で作成）
  manual.txt           このファイル
  start.bat            コードレビュー起動
  start_spec.bat       仕様レビュー起動
  sha256.txt           ハッシュ値
